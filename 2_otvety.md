# Практическая работа №1 — Ответы

## Контрольные вопросы

### 1. Что такое микроконтроллер и какие основные функциональные блоки он включает?

Микроконтроллер — это специализированная микропроцессорная система на одном кристалле, предназначенная для управления электронными устройствами. Основные функциональные блоки ATmega328P:

- **CPU (центральный процессор)** — выполняет арифметические и логические операции, обрабатывает команды из памяти программ
- **Flash-память (32 КБ)** — энергонезависимая память для хранения программы (прошивки)
- **SRAM (2 КБ)** — оперативная память для хранения переменных во время выполнения программы
- **EEPROM (1 КБ)** — энергонезависимая память для хранения данных, которые сохраняются при выключении питания
- **Порты ввода/вывода (GPIO)** — 23 линии ввода/вывода (порты B, C, D) для связи с внешними устройствами
- **Таймеры/счётчики** — Timer0, Timer1, Timer2 для генерации временных задержек, ШИМ-сигналов и подсчёта событий
- **АЦП (аналого-цифровой преобразователь)** — 10-битный, 6 каналов для преобразования аналоговых сигналов в цифровые
- **Интерфейсы связи** — USART, SPI, I2C (TWI) для обмена данными с другими устройствами
- **Система тактирования** — внутренний генератор 8 МГц и поддержка внешнего кварцевого резонатора до 20 МГц
- **Система прерываний** — 26 источников прерываний для реагирования на внешние и внутренние события

### 2. Какое назначение имеют выводы VCC, AVCC и GND микроконтроллера ATmega328P?

- **VCC (вывод 7)** — основное питание цифровой части микроконтроллера. Напряжение питания: 1.8–5.5 В (типично 5 В при работе с Arduino). Обеспечивает энергией CPU, память, порты ввода/вывода и цифровую логику.

- **AVCC (вывод 20)** — питание аналоговой части, в первую очередь аналого-цифрового преобразователя (АЦП). Должен быть подключён к VCC, даже если АЦП не используется. Для повышения точности АЦП рекомендуется подключать через LC-фильтр для подавления помех от цифровой части.

- **GND (выводы 8 и 22)** — общий провод (земля), опорный потенциал 0 В. Все напряжения в схеме измеряются относительно GND. Оба вывода GND должны быть подключены к общей шине земли для корректной работы.

**Важно:** без подключения всех трёх выводов питания микроконтроллер не будет функционировать или будет работать нестабильно.

### 3. Для чего используется токоограничивающий резистор при подключении светодиода?

Токоограничивающий резистор необходим для **защиты светодиода и вывода микроконтроллера от перегрузки по току**.

- Светодиод имеет очень малое внутреннее сопротивление (при прямом включении)
- Без резистора через светодиод потечёт слишком большой ток, что приведёт к его перегоранию
- Также может повредиться вывод микроконтроллера (максимальный ток GPIO ATmega328P — 40 мА)

**Расчёт номинала:**
```
R = (Vcc - Vled) / I
R = (5В - 2В) / 0.015А = 200 Ом ≈ 220 Ом (стандартный номинал)
```
Где: Vcc = 5 В, Vled ≈ 2 В (прямое напряжение красного LED), I = 15 мА (рабочий ток)

Типичные номиналы: **220–330 Ом** для стандартного светодиода при питании 5 В.

### 4. Какой вывод микроконтроллера ATmega328P соответствует цифровому выводу D13 платы Arduino Uno?

Цифровому выводу **D13** платы Arduino Uno соответствует вывод **PB5 (Port B, бит 5)** микроконтроллера ATmega328P — это **физический вывод 19** микросхемы в DIP-28 корпусе.

На плате Arduino Uno к этому же выводу подключён встроенный светодиод (обозначен «L» на плате), поэтому программа Blink управляет именно им без необходимости внешних подключений.

Соответствие: `pinMode(13, OUTPUT)` в Arduino IDE → регистр DDRB, бит 5 → физический вывод PB5.

### 5. Почему для моделирования в Proteus используется файл прошивки формата HEX, а не исходный код?

- **HEX-файл** — это скомпилированный машинный код в формате Intel HEX, содержащий готовые инструкции для процессора AVR
- **Микроконтроллер не понимает исходный код** (C/C++) — он исполняет только машинные команды
- Proteus эмулирует работу реального микроконтроллера, выполняя инструкции побайтно, точно как настоящий чип
- Процесс: исходный код (.ino) → компилятор avr-gcc → объектный файл → линковщик → **HEX-файл** → загрузка в Proteus
- HEX-файл не зависит от среды разработки — можно скомпилировать в Arduino IDE, Atmel Studio или командной строке

### 6. Как влияет тактовая частота микроконтроллера на выполнение программы Blink?

Тактовая частота определяет **скорость выполнения инструкций** процессора:

- При **16 МГц** (стандарт Arduino Uno): одна инструкция выполняется за 62.5 нс. Функция `delay(1000)` точно отсчитывает 1 секунду.
- При **пониженной частоте** (например, 8 МГц): процессор работает в 2 раза медленнее, но функция `delay()` автоматически компенсирует это, если частота указана правильно.
- При **неправильной настройке** частоты в Proteus (например, 1 МГц вместо 16 МГц): таймеры будут работать некорректно, и `delay(1000)` будет длиться не 1 секунду, а значительно дольше — светодиод будет мигать медленнее ожидаемого.

**Вывод:** в Proteus необходимо установить Clock Frequency = **16 MHz**, чтобы симуляция соответствовала реальной плате Arduino Uno.

### 7. В чём заключается отличие аппаратной ошибки подключения от программной ошибки?

| Параметр | Аппаратная ошибка | Программная ошибка |
|----------|-------------------|--------------------|
| **Причина** | Неправильное физическое соединение компонентов | Ошибка в исходном коде или логике программы |
| **Пример** | Светодиод подключён к PB4 вместо PB5; резистор слишком большого номинала; отсутствие GND | Указан неверный номер пина; неправильная задержка; отсутствие `pinMode()` |
| **Симптомы** | Схема не работает при правильной программе | Программа компилируется, но работает неверно |
| **Диагностика** | Проверка схемы мультиметром, визуальный осмотр | Отладка кода, анализ логики, Serial Monitor |
| **Исправление** | Перепайка, переподключение проводов | Изменение исходного кода и перекомпиляция |

**В контексте работы:** светодиод был подключён к неправильному выводу (аппаратная ошибка), при этом программа Blink была написана корректно и управляла выводом PB5.

### 8. Почему программа Blink используется как базовый тест работоспособности микроконтроллера?

Программа Blink является идеальным базовым тестом, потому что:

1. **Минимальный код** — всего несколько строк, вероятность программной ошибки минимальна
2. **Проверяет CPU** — если светодиод мигает, значит процессор выполняет инструкции
3. **Проверяет тактирование** — корректный интервал мигания подтверждает правильность Clock Frequency
4. **Проверяет GPIO** — подтверждает работоспособность порта ввода/вывода (PB5)
5. **Проверяет питание** — если схема запускается, значит VCC, AVCC и GND подключены правильно
6. **Проверяет Flash-память** — программа загружена и читается корректно
7. **Визуальная индикация** — результат виден невооружённым глазом без дополнительного оборудования

Это аналог «Hello, World!» в программировании — простейший тест, подтверждающий что вся цепочка (компиляция → загрузка → исполнение → вывод) работает корректно.

## Задания для студентов — Ход выполнения

### Задание 1. Изучить аппаратную архитектуру ATmega328P

ATmega328P — 8-битный AVR микроконтроллер фирмы Atmel (Microchip). Архитектура:
- Гарвардская архитектура (раздельные шины для программ и данных)
- RISC-процессор, 131 инструкция, большинство выполняется за 1 такт
- 32 регистра общего назначения (R0–R31)
- Рабочее напряжение: 1.8–5.5 В
- Максимальная частота: 20 МГц

### Задание 2. Создание проекта в Proteus 8

1. Запустить Proteus 8 Professional → File → New Project
2. Указать имя проекта: «Blink»
3. Schematic Design → DEFAULT → Next
4. PCB Layout → «Do not create a PCB layout» → Next
5. Firmware → «No Firmware Project» → Next
6. Summary → Finish

### Задание 3. Размещение компонентов

Через меню Component Mode (P) добавить:
- **ATmega328P** — микроконтроллер
- **LED-RED** — красный светодиод
- **RES 220** — резистор 220 Ом

### Задание 4. Подключение питания и PB5

- VCC → выводы 7 (VCC) и 20 (AVCC)
- GND → выводы 8 и 22
- PB5 (вывод 19) → резистор 220 Ом → анод LED → катод LED → GND

### Задание 5. Программа Blink в Arduino IDE

```c
void setup() {
  pinMode(13, OUTPUT);  // PB5 = D13
}

void loop() {
  digitalWrite(13, HIGH);  // Включить LED
  delay(1000);             // Ждать 1 секунду
  digitalWrite(13, LOW);   // Выключить LED
  delay(1000);             // Ждать 1 секунду
}
```

Компиляция: Sketch → Export Compiled Binary → получаем файл `.hex`

### Задание 6. Загрузка HEX в Proteus

1. Двойной клик на ATmega328P в схеме
2. Program File → указать путь к `Blink.ino.hex`
3. Clock Frequency → 16 MHz
4. OK

### Задание 7. Запуск симуляции

Нажать Run (▶). Светодиод мигает с интервалом ~1 сек → схема работает корректно.

### Задание 8. Анализ возможных ошибок

| Ошибка | Последствие | Решение |
|--------|------------|---------|
| LED подключён к неверному выводу | Не мигает | Переподключить к PB5 |
| Резистор 220 кОм вместо 220 Ом | LED не светится (ток слишком мал) | Заменить на 220 Ом |
| Не подключён AVCC | Нестабильная работа | Подключить к VCC |
| Не подключён GND | Схема не работает | Подключить оба GND |
| Неверная Clock Frequency | Неправильные задержки | Установить 16 MHz |
| HEX-файл не загружен | Нет работы программы | Указать путь к .hex |
